## 1. 创意名称 + 创意介绍

**创意名称**：GitHub Trending Daily — AI 驱动多维评价日报

**一句话概括**：AI 驱动的 GitHub 热门项目分析平台，通过 7 维健康度评分 + 大模型分析，将每日 Trending 转化为爆发布局、质量筛选、陷阱预警的三模式智能日报。

**想解决什么问题**：每天 GitHub Trending 排行榜只看 Star 数，但高 Star 不等于高质量——有些项目 Star 暴涨却 issue 堆积、长期停更、甚至权重不开源。开发者无法一眼分辨哪些是真正值得关注的好项目、哪些是热度陷阱。

**为什么会想到做这个**：我每天刷 GitHub Trending 追踪技术趋势，但经常被 Star 数误导。看了一个高星项目花半小时研究，结果发现半年没更新、社区无人维护。GitHub 明明提供了 issues/PR/contributors/releases/commits 等免费数据，却没人整合利用，于是决定用 AI 补齐这个信息差。

**产品形态**：一个 Web 应用，每日自动抓取 GitHub 热门项目，通过 7 维健康度评分 + DeepSeek 大模型分析，生成三份智能日报，支持邮件推送和浏览器交互浏览。

**核心功能**：

| 功能 | 说明 |
|------|------|
| 📊 **基础日报（6板块）** | 🧨爆发 / 🏆质量标杆 / 🌱潜力新星 / ⚠️热度陷阱 / 🤖AI雷达 / 📊数据看板 |
| 🤖 **AI 深度日报（5板块）** | 🧠模型权重 / 🔧Agent工具链 / 📊数据评测 / ⚡爆发信号 / ⚠️风险关注 |
| 🔧 **自定义日报** | 输入任意话题 → AI 自动解析 → 生成专属日报（32个话题模板 + LLM 动态板块） |
| 📧 **邮件推送** | 每日自动推送 6 板块日报到 QQ 邮箱 |
| 🎨 **Web 交互** | 深色主题、TAB 切换、秒开响应、响应式布局 |

## 2. 目标用户及痛点

**面向哪些用户**：
- 每天关注技术趋势的后端/前端/AI 开发者
- 需要技术选型评估的技术负责人和架构师
- 追踪 AI/ML 开源动态的研究者和从业者

**使用场景**：
- 早晨通勤时打开网页或邮件，3 分钟浏览今日值得关注的项目
- 技术选型时切换到「自定义模式」，输入话题生成专属日报
- AI 从业者切换到「AI 深度」TAB，只看 AI 垂类项目

**传统方式的四个痛点**：
1. **只看 Star** — GitHub Trending 只按 Star 排序，没有健康度、维护状态维度
2. **无分类视角** — 所有项目混在一起，AI / 游戏 / 金融挤在一页，没有针对性的分类推荐
3. **无法自定义** — 想了解「量化交易有什么新项目」只能手动搜索
4. **英文门槛** — 大量 README 是英文，中文开发者理解成本高

---

## 3. 价值与意义

**效率提升**：
- 传统方式每天手动刷 Trending + 逐个点开 README + 判断质量 ≈ 30 分钟/天
- 使用本产品：3 分钟看完 6 板块分类日报，每个项目都有 AI 生成的中文简介
- 自定义模式按话题生成专属日报，技术选型效率提升 10 倍

**社会价值**：
- 降低优质开源项目的发现门槛，帮助好项目获得应有的关注
- 通过「热度陷阱」板块预警高星低质项目，帮助开发者避免踩坑
- 完全开源，任何人都可以部署自己的实例

### 核心架构

```
用户浏览器（3 TAB）
  │
  ├─ TAB 1: 基础日报 ← 6板块评分引擎 → 7维健康度数据
  ├─ TAB 2: AI 深度   ← AI垂直评分引擎 → 96个AI项目分类
  └─ TAB 3: 自定义    ← 自然语言解析器 → 32话题模板 + LLM
       │
       └─ 输入"量化交易" → 规则匹配（毫秒级）→ 关键词筛选 → 日报
       └─ 输入新颖话题   → LLM解析 + json_object → 动态板块生成 → 日报
       └─ LLM失败       → 关键词降级 → 元模板兜底 → 日报
```

### 关键技术决策

1. **规则优先 + LLM 兜底** — 32 个常用话题毫秒级规则匹配，冷门话题才调 DeepSeek
2. **response_format: json_object** — 强制 LLM 输出合法 JSON，避免格式错误
3. **元模板降级** — LLM 失败时自动拼接 `{话题}热门 / {话题}新星 / {话题}精品 / 数据看板`
4. **Token 智能跳过** — 识别占位符 Token → 自动跳过 API 请求 → 优雅降级不崩溃
5. **数据库自动迁移** — 新增 7 个评分字段，`init_db()` 自动检测并追加，不丢数据
