## 一、Demo 简介

### 是什么

**GitRadar** 是一款**面向开发者的 GitHub 开源项目发现、推荐、收藏与分析工具**。它依托 GitHub 公开数据，结合热门趋势与个性化算法，每天从 500+ 项目中通过 6 维评分引擎筛选出真正有价值的项目，生成多维评价日报并推送到邮箱；同时支持自定义话题查询（接入 6 家 LLM 厂商），让开发者用一句话找到贴合个人技术偏好的仓库，实现精准、高效的开源项目发现与管理。

**产品形态**：网站系统（Flask Web 应用 + Cloudflare Tunnel 公网访问 + GitHub Pages 永久跳转）

### 面向谁

- **学生开发者**：寻找练手项目、毕业设计、入门开源仓库，避免在劣质 / 废弃项目中浪费时间
- **职场开发者**：技术选型、查找优质开源工具、框架、组件库，降低筛选成本
- **开源爱好者**：追踪全球开源趋势、挖掘新晋优质项目、跟进领域动态

### 主要功能

> 💡 **核心价值**：
> - 替代 GitHub 原生 Trending，提供**精细化筛选 + 个性化推荐**
> - 过滤劣质、废弃、无更新的**僵尸仓库**，降低筛选成本
> - 基于用户行为 / 自定义话题生成**专属推荐流**，实现千人千面
> - 一站式完成**浏览、订阅、跟进、学习管理**

#### 📊 6 板块多维评价日报

打开网站默认进入「📊 基础日报」Tab，每天自动生成 6 个维度的综合日报：

- 🧨 **正在爆发**：增长加速度最大的项目（日增量 ≥ 100）
- 🏆 **质量标杆**：健康度极高 + 社区活跃的高质量项目
- 🤖 **AI 重点项目**：大模型 / AI Agent / 具身智能垂类项目
- 📊 **编程语言分布**：热门语言趋势
- 🏷️ **连续在榜项目**：持续热门的项目追踪
- 📈 **统计摘要**：收录总数、AI 占比、爆发数等

每个项目都经过 **6 维评分**：爆发度、质量、潜力、AI 雷达、陷阱检测、综合分。已索引 **533 个项目**。

## 二、Demo 创作思路

### 灵感来源

作为开发者，我每天都会刷 GitHub Trending 找新项目，但很快发现四个真实痛点：

1. **找项目慢**：Trending 每天上新几十个项目，逐个点开 README 耗时费力
2. **筛选成本高**：GitHub 只显示 star 数，劣质、废弃、无更新的僵尸仓库混杂其中，难以辨别
3. **优质小众项目难挖掘**：热门榜单被大项目长期占据，新晋优质项目难以浮出水面
4. **找不到贴合个人技术偏好的仓库**：Trending 是综合榜单，缺乏面向个人领域的个性化推荐

### 想解决的问题

依托 GitHub 公开数据，结合热门趋势与个性化算法，精准、高效地解决开发者四大痛点：

- **找项目慢** → 6 维评分引擎每天从 500+ 项目中自动筛出 Top N，一眼看到真正有价值的项目
- **筛选成本高** → 陷阱检测维度自动识别僵尸仓库（无更新 / 无 Issue 响应 / 无 License），过滤劣质项目
- **优质小众项目难挖掘** → 爆发度维度捕捉增长加速度大的新晋项目，让小众优质项目浮出水面
- **找不到贴合个人技术偏好的仓库** → 自定义话题查询 + 多 LLM 解析，用一句话生成专属领域报告

### 为什么做这个方向

1. **替代 GitHub 原生 Trending**：提供精细化筛选 + 个性化推荐，而非纯展示
2. **刚需高频**：GitHub 是开发者最重要的资源，每天都需要
3. **技术挑战适中**：涉及爬虫、评分算法、LLM 集成、数据加密、自动化部署，技术栈丰富
4. **可量化价值**：6 维评分让项目筛选从"凭感觉"变成"看数据"
5. **差异化**：市面上的 GitHub Trending 工具只做"展示"，本作品做"评价 + 个性化 + 自动推送 + 一站式管理"

- Cloudflare Tunnel 实现本地服务公网访问（无需公网 IP，穿透 NAT）

- 一键更新脚本 `update_redirect.ps1`

#### 阶段 7：自动化部署

- GitHub Actions 每日定时任务（UTC 00:00 自动运行 Pipeline，全程无人工干预）
- Docker 容器化部署（Dockerfile + render.yaml）
- 敏感信息彻底清除（git-filter-repo 重写 90 个 commits 历史，QQ 邮箱脱敏）

### 踩坑经历

1. **端口占用导致 404**：多进程监听同一端口，请求被路由到旧实例。解决：杀掉所有进程后重启
2. **Flask f-string 双花括号转义**：CSS/JS 中的 `{}` 必须写成 `{{ }}`，否则报错
3. **Hugging Face Spaces 拒绝二进制文件**：trending.db 被 HF 拒绝。解决：改用 repos.json 回退加载
4. **Hugging Face Space 被标记为滥用**：HF 自动审核触发 "Blocked by abuse-handler by rule: Cloudflare"。解决：弃用 HF Spaces，改用 GitHub Pages 永久跳转方案
5. **Cloudflare Tunnel URL 变化**：重启后 URL 会变，旧链接失效。解决：GitHub Pages 跳转页 + 一键更新脚本
6. **git 历史中残留敏感信息**：QQ 邮箱在历史 commit 中。解决：用 git-filter-repo 重写全部 90 个 commits 历史彻底清除

## 六、项目结构

```
github-trending-daily/
├── src/
│   ├── fetcher/           # 数据抓取（Trending + Search API + GraphQL）
│   ├── processor/        # 数据处理（去重、分类、6 维评分、AI 分析、中文描述、自定义解析）
│   ├── storage/          # SQLite + JSON 双存储
│   ├── reporter/         # 日报生成（HTML + Markdown + AI 报告 + 自定义报告）
│   ├── notifier/         # 邮件推送 + URL 监控守护 + URL 变更通知
│   └── utils/            # 加密工具（AES + HMAC + HTML 转义）
├── web/
│   └── app.py            # Flask Web 服务（5 个 Tab + 架构可视化）
├── .github/workflows/    # GitHub Actions 每日定时任务
├── Dockerfile            # 容器化部署
├── render.yaml           # Render.com 部署配置
├── index.html            # GitHub Pages 永久跳转页
├── redirect.json         # 实时服务地址配置
└── update_redirect.ps1   # 一键更新跳转地址脚本
```
